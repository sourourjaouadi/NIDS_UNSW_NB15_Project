## What is this project?

A machine learning system that monitors network traffic and automatically detects whether a connection is **normal or malicious** — and if malicious, **identifies the type of attack**. Built on the UNSW-NB15 benchmark dataset using a two-stage classification pipeline.

---

## Two-Stage Pipeline

**Stage 1 — Is it an attack?**
Every network flow is first classified as either normal traffic or an intrusion attempt.

**Stage 2 — What kind of attack?**
If an attack is detected, a second model identifies the specific attack category.

---

## Dataset — UNSW-NB15

Generated by the Australian Centre for Cyber Security in a real cyber range environment. It contains realistic network traffic mixed with synthetic attack scenarios.

| | |
|---|---|
| Total records | 2,540,044 |
| Features | 43 |
| Attack categories | 9 |

### Attack Types Covered

| Attack | Description |
|---|---|
| **DoS** | Floods the target to make services unavailable |
| **Fuzzers** | Sends malformed inputs to crash applications |
| **Exploits** | Takes advantage of known software vulnerabilities |
| **Reconnaissance** | Scans the network to gather information |
| **Backdoor** | Establishes hidden persistent access |
| **Generic** | Brute-force and cryptographic attacks |
| **Analysis** | Probes system behavior to find weaknesses |
| **Shellcode** | Injects malicious code for remote execution |

## Models Used

### 🌲 Random Forest
An ensemble of 600 decision trees that vote on each classification. Uses balanced class weighting to handle the imbalance between normal and attack traffic. Provides built-in out-of-bag validation.

### ⚡ XGBoost
Gradient boosted trees with regularization and early stopping. Particularly strong on tabular data and handles imbalanced classes well.

## Results

### Binary Detection (Normal vs Attack)

| Model | Accuracy | Recall | F1-Score | ROC-AUC |
|---|---|---|---|---|
| 🌲 Random Forest | **99.21%** | **99.63%** | **99.25%** | **99.81%** |
| ⚡ XGBoost | 99.14% | 99.31% | 99.16% | 99.77% |
|



> Random Forest achieves the best results across both tasks. Recall is prioritized over precision — in security, missing an attack is more costly than a false alarm.

## Explainable AI — SHAP

To make the models transparent and trustworthy, we apply SHAP (SHapley Additive exPlanations) to explain both global feature importance and individual predictions.

### Most Important Features

| Rank | Feature | Meaning |
|---|---|---|
| 1 | `sttl` | Source TTL value — attackers often use non-standard values |
| 2 | `ct_state_ttl` | TTL-based state counter |
| 3 | `dttl` | Destination TTL |
| 4 | `sbytes` | Number of bytes sent by source |
| 5 | `dbytes` | Number of bytes sent by destination |
| 6 | `dur` | Flow duration — attacks tend to be very short or very long |
| 7 | `spkts` | Source packet count — DoS shows extremely high counts |

## Data Preprocessing

- Merged 4 raw CSV files into a single dataset
- Removed duplicate rows and placeholder values
- Dropped columns that cause data leakage (IP addresses, timestamps)
- Encoded categorical features (protocol, service, connection state)
- Split into **70% training / 15% validation / 15% test** with stratification
- Applied balanced class weighting to handle the normal/attack imbalance
